# Performance — 全言語共通ルール

## 計測ファースト

- 推測でパフォーマンス改善をしない
- プロファイラーで実測してからボトルネックを特定する
- 改善前後の数値を記録する

## 一般原則

### アルゴリズムとデータ構造

- O(n²) 以上のアルゴリズムを大量データに使わない
- 適切なデータ構造を選ぶ（検索には Map/Set、順序には List）
- ループ内での不必要なオブジェクト生成を避ける

### ネットワーク

- 必要なデータだけ取得する（GraphQL / 絞り込み API）
- ページネーション・無限スクロールを使う
- キャッシュを適切に活用する（HTTP Cache / メモリキャッシュ）
- 並列リクエストを活用する（`Future.wait` / `Promise.all`）

### データベース

- N+1 クエリを避ける
- 検索カラムにインデックスを付ける
- バッチ操作を活用する（まとめて insert/update）

### メモリ

- 大きなオブジェクトのライフサイクルを管理する
- リソース（ストリーム・コントローラー）は使い終わったら閉じる
- メモリリークを DevTools / Profiler で定期的に確認する

## Flutter 固有（→ rules/flutter/ を参照）

- `const` Widget の徹底
- `build()` 内での重い処理の禁止
- `ListView.builder` の使用
