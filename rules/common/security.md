# Security — 全言語共通ルール

## 絶対禁止

- APIキー・パスワード・トークンをコードにハードコード
- 機密情報をバージョン管理に含める（`.env` を commit しない）
- ユーザー入力を未検証のまま使用（SQL injection / XSS / Command injection）
- HTTP（非HTTPS）での認証情報の送信
- ログへの個人情報・認証情報の出力

## 入力検証

```
# すべての外部入力を信頼しない
- APIレスポンス
- ユーザー入力フォーム
- URLパラメータ
- ファイル内容
- 環境変数

# 検証項目
- 型の確認
- 長さの制限
- 許可された文字セット
- ビジネスルールの妥当性
```

## 認証・認可

- トークンは `flutter_secure_storage` / `Keychain` / `Keystore` に保存
- セッション有効期限を設定する
- 権限チェックはサーバーサイドで必ず行う（クライアントのみは不可）
- JWT の検証は署名・有効期限・発行者を確認する

## 依存関係

- 使用パッケージのセキュリティアドバイザリを定期的に確認
- 未メンテナンスのパッケージを使わない（最終更新から1年以上）
- `pub outdated` で脆弱性のある古いバージョンを更新する

## エラーレスポンス

- スタックトレースをユーザーに見せない
- 内部実装の詳細をエラーメッセージに含めない
- 認証エラーは「ユーザー名またはパスワードが間違っています」（どちらが間違いか教えない）
