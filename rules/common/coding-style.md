# Coding Style — 全言語共通ルール

## 命名規則

- 意図が伝わる名前をつける（`data`, `temp`, `obj` は禁止）
- 略語は広く知られているもののみ許可（`id`, `url`, `api`）
- ブール変数は `is`, `has`, `can`, `should` で始める
- コレクションは複数形

## コード構造

- 関数は1つのことだけを行う（単一責任）
- 関数の行数: 目安 30行以内
- ネストは3階層まで（それ以上は関数に抽出）
- マジックナンバーは定数に置き換える

## コメント

- WHY を書く（WHAT はコードが語る）
- TODO は `TODO(name): 理由` の形式
- 古いコメントは削除する（コードと矛盾するコメントは害悪）

## エラーハンドリング

- エラーを握りつぶさない（空の catch は原則禁止）
- エラーメッセージは具体的に（「エラーが発生しました」は不可）
- ユーザー向けと開発者向けのエラーを分ける

## セキュリティ基本

- 機密情報をコードにハードコードしない
- ユーザー入力は常にバリデーション
- ログに個人情報・認証情報を出力しない
